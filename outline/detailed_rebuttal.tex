	\item Review A:

	We removed 'classic'
	
	\item \red{sub-linear in what? How is the input size measured?}
	
	We added a footnote at line 78, page 2.

\item Review B:
	\begin{itemize}
		\item \red{Why is the diagram on p.25 not covering all the abstract machines in the paper?}
	
	Now it does cover all machines but for the HAM, which is really just a technical tool not deserving to appear in the conclusions. The diagram is also at the end of the introduction, now.		
\end{itemize}


\item Review C:
	\begin{itemize}
		\item \red{No example showing duplication.}
	
		At the end of the technical report associated to the paper there is a simple example of duplicating term which is evaluated with each one of the studied machines, to show how they differently deal with it.
		
		After introducing each machine we in the paper now we point to the technical report saying that there the reader can find an example of duplication.
		
		\item \red{Definition 6.3 requires s'' to "surround l". It seems that it must be the case after a bt2,l step. Why do you add this requirement?}
		
		In the rebuttal we answered that the requirement was indeed not needed. But we were wrong. It is true that whatever bt2,l steps would extend the context and the log, but the definition of 'surrounding' additionally asks the tape to be empty, which is required.
		
		\item The reviewer pointed out that we used twice the dichotomy \emph{local/global}, once for to contrast the management of logs by the lambda-IAM and the lambda-JAM, and once for contrasting the way lambda-JAM and the lambda-PAM trace the history of the run. We changed the second one, replacing local/global with \emph{fragmented/monolitic} 
\end{itemize}
\end{itemize}
